
# Release Notes — Version 0.5.1

## Overview

Version 0.5.1 marks the beginning of active development following the restoration of the legacy SudokuSolver project.

This release focuses on restoring a stable development environment rather than introducing new solving techniques.

The objective is to establish a clean foundation upon which future development can proceed with confidence.

---

## Highlights

### Project Revival

* Restored the original Git repository.
* Re-established active development.
* Preserved historical design notes and documentation.

### Build System

* Added a development Makefile.
* Standardized common development tasks.
* Established `make check` as the primary project health check.

### Modernization

* Restored successful syntax checking under Perl 5.34.
* Updated legacy Moose type definitions.
* Replaced obsolete debugging dependency on Data::Dump with Data::Dumper.
* Corrected legacy module loading issues.

### Testing

* Established the project test framework.
* Added the initial placeholder test.
* Prepared the project for incremental test-driven development.

---

## Existing Solver Techniques

The current solver includes implementations or partial implementations of:

* Naked Singles
* Hidden Singles
* Pointing / Claiming
* Naked Pairs
* Hidden Pairs
* X-Wings
* Experimental Remote Pairs

These techniques are being preserved while the surrounding infrastructure is
modernized.

---

## Known Limitations

Current areas targeted for future work include:

* Command-line interface improvements.
* Separation of solving logic from the executable.
* Refactoring Grid.pm into smaller modules.
* Expanded regression testing.
* Improved documentation.
* Consistent strategy interfaces.
* Explainable solving steps.

---

## Looking Ahead

The immediate development priorities are:

1. Improve command-line handling.
2. Expand automated testing.
3. Move solver orchestration into Solver.pm.
4. Continue modularizing the solver.
5. Preserve existing behavior while improving maintainability.

Long-term development will focus on creating an explainable, logic-based Sudoku
solver that can both solve puzzles and teach the reasoning behind each
deduction.
