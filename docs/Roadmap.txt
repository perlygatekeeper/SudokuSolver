
# Version 0.5.1

## Guiding Principles

These principles guide the long-term evolution of SudokuSolver.

* Preserve backward compatibility whenever practical.
* Prefer logic-based solving over brute force.
* Use accepted Sudoku technique names in code and public docs.
* Preserve original/historical project terminology in notes.
* Add tests before major refactors.
* Keep each solving strategy small, explainable, and independently testable.

## Version 0.5.1 - Revival Cleanup

Goals:

* Restore the project to a clean, buildable development state.
* Add a Makefile for common development commands.
* Add the initial test harness.
* Preserve historical notes and solution-output examples.
* Begin documenting the existing solver capabilities.
* Prepare for moving solve orchestration into Solver.pm.

Completed techniques so far:

* Singletons / Naked Singles
* Lone Representatives / Hidden Singles
* Imaginary Values / Pointing and Claiming
* Naked Pairs
* Hidden Pairs
* X-Wings
* Experimental Remote Pairs notes/code

Cleanup activities:

* Fix Makefile typos and target issues.
* Add or improve .gitignore.
* Add t/00_placeholder.t.
* Add syntax/test/check automation.
* Document current output style.
* Identify unfinished strategy work.
* Prepare command-line interface cleanup.

## Version 0.5.2 - CLI and Harness
