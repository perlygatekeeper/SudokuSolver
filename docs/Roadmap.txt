# Roadmap

## SudokuSolver Development Roadmap

---

## Project Philosophy

SudokuSolver is intended to be a logic-based Sudoku solver whose primary goal
is to solve puzzles using techniques that a human player can understand.

Whenever practical:

* Prefer logical deduction over brute-force search.
* Keep solving techniques independent and easily testable.
* Each solving strategy should be capable of explaining why a deduction is valid.
* Preserve backward compatibility whenever practical.
* Favor readable, maintainable code over clever code.
* Keep commits small and focused on a single logical change.
* Follow test-driven development whenever possible.

Ultimately, SudokuSolver should become not only an excellent solver, but an
excellent teaching tool.

## Long-Term Architecture

Cell
    Represents a single Sudoku cell.

Grid
    Represents puzzle state and provides lookup/mutation services.

Solver
    Controls solving order and applies deductions.

Strategy
    Finds logical deductions.

Deduction
    Represents a logical action.

Statistics
    Records solving metrics.

Difficulty
    Rates puzzles based on a versioned interpretation of solve statistics.

Hint
    Presents the next logical deduction.

Explain
    Generates human-readable reasoning for deductions.

Benchmark
    Measures solver capability against stable puzzle collections.

Formatter / Renderer
    Produces console, HTML, GUI, Markdown, or other output.

## Development Guidelines

Every commit should ideally:

* Perform one logical task.
* Compile successfully.
* Pass `make check`.
* Avoid trailing whitespace.
* Be accompanied by documentation updates when appropriate.
* Leave the repository in a releasable state.

Release checklist:

```bash
make PERL=perl check
git diff --check
make PERL=perl benchmark
```


Uniqueness Strategies
---------------------

    [x] Unique Rectangle Type 1
    [x] Unique Rectangle Type 2

These strategies rely on the puzzle having a unique solution. Additional
Unique Rectangle forms should be added individually with focused positive and
negative tests.


Empty Rectangle
---------------

    [x] Empty Rectangle

Implemented the box-based Empty Rectangle pattern with external row and column
strong links, structured deductions, human-readable explanations, positive and
negative tests, and benchmark contribution reporting.

Simple Coloring
---------------

    [x] Shared strong-link graph infrastructure
    [x] Color Trap
    [x] Color Wrap

Added Sudoku::StrongLinks to discover row, column, and box conjugate pairs,
build candidate graphs, and return connected components. Added Simple Coloring
with alternating two-color assignment, color-trap eliminations, color-wrap
eliminations, structured deductions, human-readable explanations, positive and
negative tests, and strategy and algorithm documentation.


X-Chains
--------

    [x] Alternating strong/weak same-digit chain search
    [x] Common-peer endpoint eliminations

Added X-Chains using the shared Sudoku::StrongLinks graph infrastructure.
The strategy searches bounded alternating paths that begin and end with strong
links, emits structured deductions with human-readable chain notation, and
includes focused positive and negative tests plus strategy and algorithm
documentation.
