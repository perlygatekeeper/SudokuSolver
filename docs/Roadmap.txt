# Roadmap

## SudokuSolver Development Roadmap

---

## Project Philosophy

SudokuSolver is intended to be a logic-based Sudoku solver whose primary goal is to solve puzzles using techniques that a human player can understand.

Whenever practical:

* Prefer logical deduction over brute-force search.
* Keep solving techniques independent and easily testable.
* Each solving strategy should be capable of explaining *why* a deduction is valid.
* Preserve backward compatibility whenever practical.
* Favor readable, maintainable code over clever code.
* Keep commits small and focused on a single logical change.
* Follow test-driven development whenever possible.

Ultimately, SudokuSolver should become not only an excellent solver, but an excellent teaching tool.

Long-Term Vision
==============================================================================

Cell
    Represents a single Sudoku cell.

Grid
    Represents puzzle state and provides lookup/mutation services.

Solver
    Controls solving order and applies deductions.

Strategy
    Finds logical deductions.

Deduction
    Represents a logical action.

Formatter / Renderer
    Produces console, HTML, GUI, Markdown, or other output.

Statistics
    Records solving metrics.

Difficulty
    Rates puzzles based on the solving process.

Hint
    Presents the next logical deduction.

Explain
    Generates human-readable reasoning for every deduction.

## Development Guidelines

Every commit should ideally:

* Perform one logical task.
* Compile successfully.
* Pass `make check`.
* Be accompanied by documentation updates when appropriate.
* Leave the repository in a releasable state.
