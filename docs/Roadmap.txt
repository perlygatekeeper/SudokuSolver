# Roadmap

## SudokuSolver Development Roadmap

---

## Project Philosophy

SudokuSolver is intended to be a logic-based Sudoku solver whose primary goal is to solve puzzles using techniques that a human player can understand.

Whenever practical:

* Prefer logical deduction over brute-force search.
* Keep solving techniques independent and easily testable.
* Each solving strategy should be capable of explaining *why* a deduction is valid.
* Preserve backward compatibility whenever practical.
* Favor readable, maintainable code over clever code.
* Keep commits small and focused on a single logical change.
* Follow test-driven development whenever possible.

Ultimately, SudokuSolver should become not only an excellent solver, but an excellent teaching tool.

## Deduction Objects

Future strategy modules should return structured Deduction objects rather than
only reporting progress or printing directly.

A Deduction object should describe:

* strategy name
* action performed
* affected cell or cells
* affected candidate or solved value
* reason the deduction is valid
* optional human-readable explanation

Example:

{
    strategy    => 'Hidden Single',
    action      => 'set_value',
    row         => 3,
    column      => 7,
    value       => 5,
    explanation => 'Candidate 5 appears only once in row 3.'
}

This design supports:

* Hint Mode
* Explain Mode
* difficulty rating
* strategy statistics
* regression testing
* GUI or web visualization
* cleaner separation between solving logic and output formatting

Long-term solving flow:

Grid
  -> Strategy
  -> Deduction
  -> Solver
  -> Explanation Log
  -> Output / Hint / Report
