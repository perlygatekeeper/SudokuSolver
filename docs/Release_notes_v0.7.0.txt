==============================================================================
SudokuSolver v0.7.0 Release Notes
==============================================================================

Release Date: 2026-07-10

This release completes Phase 2 of the SudokuSolver modernization effort.
The project now exposes user-facing deduction features on top of the
strategy and deduction engine introduced in earlier releases.

Highlights
==========

User-Facing Solver Features
---------------------------

* Added solve statistics.
* Added step-by-step solving.
* Added Hint Mode.
* Added Explain Mode.
* Added versioned difficulty ratings.
* Added contradiction detection.

Puzzle Loading
--------------

* Improved puzzle-string normalization.
* Whitespace is treated as formatting.
* Dots, underscores, dashes, and zeroes may be used for empty cells.
* Files may contain one puzzle per line.
* Files may also contain a single puzzle spread across nine rows.
* End-of-line comments are supported in puzzle files.


Example Puzzles
---------------

Three documented example puzzles are provided under:

    Puzzles/Examples/solved.sdk
    Puzzles/Examples/stalled.sdk
    Puzzles/Examples/contradiction.sdk

They demonstrate the three terminal solver states introduced and clarified in
this release. Run all three with:

    make examples

Benchmarking
------------

* Added Sudoku::Benchmark.
* Added the --benchmark command-line option.
* Added make benchmark and make benchmark-first50 targets.
* Established the first 50 canonical 17-clue puzzles as the primary
  capability benchmark.

Current Benchmark Goal
----------------------

Canonical 17-clue benchmark, first 50 puzzles:

    Current known baseline: 41 / 50 solved
    Goal:                  50 / 50 solved without guessing

Future strategy work should improve this benchmark without introducing
regressions.

Architecture
============

The solver now supports the following public engine features:

* run
* step
* hint
* explain
* difficulty
* statistics
* contradiction status
* benchmark execution

Difficulty ratings are versioned and derived from solve statistics, allowing
future rating algorithms to re-evaluate old solve data.

Two-String Kite
===============

Added the Two-String Kite strategy, including row/column strong-link detection,
box-connected string validation, structured deductions, human-readable
explanations, positive and negative tests, and strategy documentation.

Empty Rectangle
===============

Added the Empty Rectangle strategy in both row-link and column-link forms.
The implementation recognizes the empty box intersection and two candidate
arms, validates an external conjugate pair, and emits structured candidate-
removal deductions with human-readable explanations. Added positive and
negative tests plus strategy and algorithm documentation. Difficulty rating
method version is now 1.7.

Simple Coloring
===============

Added reusable Sudoku::StrongLinks infrastructure for row, column, and box
conjugate pairs, candidate graphs, and connected components. Added the Simple
Coloring strategy with Color Trap and Color Wrap deductions, structured and
human-readable explanations, positive and negative tests, and strategy and
algorithm documentation. Difficulty rating method version is now 1.8.


X-Chains
========

Added the X-Chains strategy for same-digit alternating inference paths. The
implementation reuses Sudoku::StrongLinks, alternates strong and weak links,
removes candidates that see both chain endpoints, records structured and
human-readable explanations, and includes positive and negative tests plus
strategy and algorithm documentation. Difficulty rating method version is now
1.9.

Multi-Coloring
==============

Added the Multi-Coloring strategy for interactions between separate colored
strong-link components. The implementation supports color-collision and
color-wing eliminations, shares component coloring with Simple Coloring, emits
structured human-readable deductions, and includes focused positive and
negative tests plus strategy and algorithm documentation. Difficulty rating
method version is now 2.0.


XY-Chains
=========

Added the XY-Chains strategy for alternating implications through visible
bivalue cells. The implementation searches bounded non-repeating paths,
removes the shared endpoint candidate from common peers, emits structured
human-readable deductions, and includes focused positive and negative tests
plus strategy and algorithm documentation. Difficulty rating method version
is now 2.1.

Patch 0063 adds Unique Rectangle Types 3 and 4. Type 3 recognizes the roof
extras as a virtual naked-subset cell; Type 4 uses a roof strong link to remove
the other deadly-pair candidate. Both strategies include human-readable
explanations and separate benchmark accounting. Difficulty rating method 2.2
adds active scores for the new strategies.

Advanced strategies
-------------------

* Added Alternating Inference Chains (AIC) with candidate-node strong/weak links.
* Stalled command-line output now includes an 81-character puzzle-state string.
